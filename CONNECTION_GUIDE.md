# 🛠️ 구글 시트 연동 및 테스트 상세 가이드 (Step-by-Step)

사용자가 직접 구글 시트 데이터를 웹서비스에 연결하고 테스트하는 방법입니다.

---

### STEP 1. 구글 시트 데이터 준비
1. 새로운 구글 스프레드시트를 만듭니다.
2. **첫 번째 행(A1, B1, C1)**에 반드시 아래 항목명을 입력하세요 (대소문자 구분):
   - `enochMonth` | `enochDay` | `feast`
3. **두 번째 행**부터 테스트 데이터를 입력합니다:
   - 예: `1` | `14` | `유월절 테스트`

---

### STEP 2. Apps Script 배포 (연결 다리 만들기)
1. 시트 상단 메뉴: **확장 프로그램 > Apps Script** 클릭
2. 열린 창에 프로젝트 루트의 `apps-script-template.js` 내용을 전체 복사하여 붙여넣습니다.
3. 상단 **[배포]** 버튼 > **[새 배포]** 클릭
4. **유형 선택**: [웹 앱] (톱니바퀴 아이콘)
5. **설정**:
   - 설명: `에녹달력 API`
   - 다음 사용자로 실행: `나`
   - 액세스 권한이 있는 사용자: **[모든 사용자]** (매우 중요!)
6. **[배포]** 버튼 클릭 후, 보안 승인 절차를 거칩니다.
7. 완료 화면의 **웹 앱 URL**을 복사합니다.

---

### STEP 3. 웹서비스에 연결 및 테스트
1. 프로젝트 폴더에 `.env.local` 파일을 만듭니다.
2. 복사한 URL을 아래 형식으로 입력하고 저장합니다:
   ```env
   NEXT_PUBLIC_SHEET_API_URL=여러자의_웹_앱_URL
   ```
3. 터미널에서 서버를 실행합니다:
   ```bash
   npm run dev
   ```
4. 브라우저에서 `localhost:3000` 접속 후, 달력 상단의 **새로고침 아이콘(파란색)**이 나타나며 시트 데이터가 반영되는지 확인하세요.

---

### 🧪 테스트 확인 포인트
- 시트에서 `feast` 내용을 수정하고 다시 로드했을 때 달력에 즉시 반영되는가?
- 존재하지 않는 날짜(예: 13월) 입력 시 에러 없이 기본 달력이 잘 나오는가?
